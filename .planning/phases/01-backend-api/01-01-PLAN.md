# Plan 01-01: Backend API — Server Setup, Fetch, Cleanup, Checkout Detection & Export

## Objective

Implement the complete backend API in server.js with four routes:
- POST /api/fetch — fetch a VSL URL, clean HTML, detect checkout links
- POST /api/export — inject affiliate pixel/player, replace checkout links, return downloadable HTML

## Context

- Tech stack: Node.js + Express + axios + cheerio
- Structure: server.js + public/index.html + package.json
- Runtime: local, port 3000

## Tasks

### Task 1: Project initialization (package.json + dependencies)
type: auto

Initialize package.json and install dependencies: express, axios, cheerio.

Done: package.json exists, node_modules installed.

### Task 2: Server bootstrap (server.js skeleton)
type: auto

Create server.js with Express app, CORS headers, JSON body parser, and static file serving from public/.

Done: server starts on port 3000, GET / returns 200.

### Task 3: POST /api/fetch route
type: auto

Implement fetch route:
- Accept { url }
- Use axios with anti-bot headers (User-Agent Chrome, Accept, Accept-Language, Cache-Control)
- maxRedirects: 10
- Parse HTML with cheerio
- Run cleanup pipeline (CLEAN-01 through CLEAN-08)
- Run checkout detection (CHECK-01 through CHECK-04)
- Return { html, summary: { scriptsRemoved, vslDetected, checkoutLinks } }

Done: route exists, cleanup removes tracking scripts, checkout links detected.

### Task 4: POST /api/export route
type: auto

Implement export route:
- Accept { html, headerPixel, headerPreload, vslembed, checkoutLinks }
- Inject headerPixel + headerPreload before </head>
- Replace <!-- [VSL_PLACEHOLDER] --> div with vslembed
- Replace each checkout link href with affiliate URL
- Return HTML file download as "pagina-afiliado.html"

Done: route returns downloadable HTML with all substitutions.

## Success Criteria

1. POST /api/fetch with a real VSL URL returns cleaned HTML and summary
2. Cleaned HTML has no Facebook Pixel, GTM, VTURB player scripts, or tracking iframes
3. POST /api/export returns downloadable HTML with affiliate substitutions applied
4. Checkout links are classified by platform and bundle context
