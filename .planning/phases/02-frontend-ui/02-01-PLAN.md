Fill in the `<script>` block at the bottom of index.html (before `</body>`) with all

-- STATE --
```js
const state = {
  fetchedHtml: null,        // string — the cleaned HTML from /api/fetch
  checkoutLinks: [],        // Array<{ href, selector, anchorText, platform, bundle }>
                            // populated from summary.checkoutLinks on fetch
};
```

-- HELPER: setLoading(btn, isLoading, defaultLabel) --
- If isLoading: btn.disabled = true, btn.classList.add('loading'),
  btn.innerHTML = '<span class="spinner"></span>' + loadingLabel
  (fetch btn: "Extraindo...", export btn: "Gerando...")
- If not: btn.disabled = false, btn.classList.remove('loading'), btn.innerHTML = defaultLabel

-- HELPER: showError(message) --
- Get #error-banner, set its textContent to message, set display to 'block'

-- HELPER: hideError() --
- Set #error-banner display to 'none', textContent to ''

-- FUNCTION: renderCheckoutInputs(links) --
Receives Array<{ href, selector, anchorText, platform, bundle }>.
Gets #checkout-inputs element and replaces its innerHTML.

If links.length === 0:
  Render 3 .checkout-item rows with labels "Bundle 2 Potes", "Bundle 3 Potes", "Bundle 6 Potes"
  and empty text inputs (no data-selector attribute needed for fallback inputs).

If links.length > 0:
  For each link, build a .checkout-item:
  - Label text:
    - If link.bundle !== null: "Bundle ${link.bundle} Potes"
    - Else if link.anchorText (trimmed, non-empty): link.anchorText (truncated to 40 chars)
    - Else: link.platform
    Append a .platform-tag span inside the label showing the platform name.
  - Input: type text, value = link.href || '', full width, class 'checkout-url-input',
    data-selector="${link.selector}" (used by exportPage to build the checkoutLinks array)

-- FUNCTION: fetchPage() --
1. hideError()
2. Get url = #url-input value trimmed. If empty: showError("Por favor, insira uma URL válida.") and return.
3. setLoading(fetchBtn, true)
4. Hide #summary-card (display none)
5. Call fetch('/api/fetch', { method:'POST', headers:{'Content-Type':'application/json'}, body: JSON.stringify({ url }) })
6. On network error: showError("Erro de rede: não foi possível conectar ao servidor."), setLoading(fetchBtn, false), return
7. Parse response as JSON. If response.ok is false: showError(data.error || "Erro ao extrair página."), setLoading(fetchBtn, false), return
8. Store: state.fetchedHtml = data.html; state.checkoutLinks = data.summary.checkoutLinks;
9. Populate #summary-card innerHTML:
   ```
   ✅ Página extraída &nbsp;|&nbsp;
   🗑️ ${data.summary.scriptsRemoved} scripts removidos &nbsp;|&nbsp;
   🎬 Player VSL detectado: ${data.summary.vslDetected ? 'Sim' : 'Não'} &nbsp;|&nbsp;
   🔗 ${data.summary.checkoutLinks.length} links de checkout encontrados
   ```
   Set #summary-card display to 'block'
10. Call renderCheckoutInputs(data.summary.checkoutLinks)
11. Reveal sections: #section-header, #section-player, #section-checkout, #section-export
    Set each to display 'block'
12. setLoading(fetchBtn, false) — restore label to "Extrair Página"
13. Scroll #section-header into view smoothly

-- FUNCTION: exportPage() --
1. If state.fetchedHtml is null: showError("Extraia uma página primeiro.") and return
2. hideError()
3. setLoading(exportBtn, true)
4. Collect form values:
   - headerPixel = #header-pixel textarea value
   - headerPreload = #header-preload textarea value
   - vslembed = #vsl-embed textarea value
   - checkoutLinks = Array.from(document.querySelectorAll('.checkout-url-input'))
       .map(input => ({ selector: input.dataset.selector, affiliateHref: input.value.trim() }))
       .filter(link => link.selector && link.affiliateHref)
5. Build requestBody = { html: state.fetchedHtml, headerPixel, headerPreload, vslembed, checkoutLinks }
6. Call fetch('/api/export', { method:'POST', headers:{'Content-Type':'application/json'}, body: JSON.stringify(requestBody) })
7. On network error: showError("Erro de rede ao gerar arquivo."), setLoading(exportBtn, false), return
8. If response.ok is false:
   Try to parse JSON error message. showError(data.error || "Erro ao gerar página."), setLoading(exportBtn, false), return
9. On success:
   - const blob = await response.blob()
   - const blobUrl = URL.createObjectURL(blob)
   - const a = document.createElement('a'); a.href = blobUrl; a.download = 'pagina-afiliado.html'
   - document.body.appendChild(a); a.click(); document.body.removeChild(a)
   - setTimeout(() => URL.revokeObjectURL(blobUrl), 5000)
10. setLoading(exportBtn, false)

-- EVENT WIRING --
After function definitions, add:
```js
const fetchBtn  = document.getElementById('fetch-btn');
const exportBtn = document.getElementById('export-btn');
const urlInput  = document.getElementById('url-input');

fetchBtn.addEventListener('click', fetchPage);
exportBtn.addEventListener('click', exportPage);

// Allow Enter key in URL input to trigger fetch
urlInput.addEventListener('keydown', (e) => {
  if (e.key === 'Enter') fetchPage();
});
```

IMPORTANT: The fetchedHtml string must NEVER be placed in any DOM element's value, innerHTML,
or data attribute. It lives only in state.fetchedHtml until exportPage() POSTs it.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → /api/fetch | URL input crosses to server — already validated server-side (SSRF protection in server.js) |
| /api/fetch → browser | Large HTML payloads returned to client — stored in JS memory, not rendered to DOM |
| user → /api/export | User-supplied affiliate URLs written into downloaded file — user is the author, no third-party trust issue |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | state.fetchedHtml | mitigate | Never place fetchedHtml in DOM (innerHTML, value, data-*). Kept in JS memory only until POST. |
| T-02-02 | Information Disclosure | checkout URL inputs | accept | Inputs contain affiliate URLs entered by the user themselves — no sensitive third-party data exposed |
| T-02-03 | Spoofing | fetch() calls | mitigate | Both API calls target relative paths (/api/fetch, /api/export) — no cross-origin requests, no CORS issues |
| T-02-04 | Denial of Service | Large HTML in export body | accept | Server already enforces 10MB Content-Length limit; client sends back what it received |
</threat_model>
