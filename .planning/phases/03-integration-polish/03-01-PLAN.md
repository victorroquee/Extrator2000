## The 5 target patterns (exact real-world shapes)

Pattern 1 — VTURB embed div (must be REMOVED, VSL_PLACEHOLDER injected at position):
```html
<div class="video-container">
  <div id="vid_6712abc1def23456789012ab">
    <script src="https://cdn.vturb.com/player/p/6712abc1def23456789012ab.js"></script>
  </div>
</div>
```
Note: The outer div.video-container triggers CLEAN-06 wrapper removal path.
The inner script triggers CLEAN-01 (keyword "vturb") and sets vslDetected=true.
Because vslDetected is true before the iframe loop runs, and there is no iframe here,
the placeholder is injected via the "body-prepended" fallback path. Verify this works.

Pattern 2 — ClickBank hop link (must be DETECTED, platform=ClickBank, bundle=6):
```html
<a href="https://acme.hop.clickbank.net/?affiliate=myid&cbpage=sales&qty=1">
  Comprar Agora — 6 Potes
</a>
```

Pattern 3 — Meta Pixel script + noscript pair (BOTH must be REMOVED):
```html
<script>
!function(f,b,e,v,n,t,s){if(f.fbq)return;n=f.fbq=function(){};
fbq('init','1234567890123456');fbq('track','PageView');
</script>
<noscript><img height="1" width="1" style="display:none"
  src="https://www.facebook.com/tr?id=1234567890123456&ev=PageView&noscript=1"/></noscript>
```
CLEAN-01 removes the script (content includes "fbq").
CLEAN-03 removes the noscript (content includes "facebook").

Pattern 4a — VTURB preload link (must be REMOVED via CLEAN-04):
```html
<link rel="preload" href="https://cdn.vturb.com/player/p/abc123.js" as="script">
```

Pattern 4b — VTURB window preload script (must be REMOVED via CLEAN-01, keyword "vturb"):
```html
<script>window.vturb_preload = { vid: "6712abc1def23456789012ab", autoplay: true };</script>
```

Pattern 5 — Hotmart pay link (must be DETECTED, platform=Hotmart, bundle=3):
```html
<a href="https://pay.hotmart.com/ABCD1234E?checkoutMode=10&bid=1709123456789">
  Quero o Pacote Popular — 3 Potes
</a>
```

## Known gap to investigate

The VTURB embed pattern (Pattern 1) does NOT use an iframe — it uses a bare
`<script src="cdn.vturb.com/...">` inside a `<div id="vid_...">`. Current server.js
logic removes the script (CLEAN-01) and sets vslDetected=true, but the outer div
remains in the DOM. The placeholder is then prepended to body instead of replacing
the div. This may leave an empty `<div id="vid_...">` and `<div class="video-container">`
in the output. Investigate and fix so the wrapper div is removed and the placeholder
is injected in its place.
