## Structural Layout

### Section card: `#section-extra-scripts`

Follows the `section-hidden` + `showSection()` pattern from phases 4 and 5.

```html
<section class="card section-hidden" id="section-extra-scripts">
  <p class="card-title">C.3 — Scripts Extras</p>
  <div id="scripts-list-container">
    <!-- Dynamically populated by JS — empty on init -->
  </div>
  <button class="btn btn-primary" id="btn-add-script" type="button"
          style="margin-top: 12px; padding: 8px 16px; font-size: 0.85rem;">
    + Adicionar Script
  </button>
</section>
```

Position in DOM: after `#section-header` (B — Scripts de Header do Afiliado) and before `#section-player` (C — Embed do Player VTURB). This matches decision D-04.

Primary visual anchor: `#btn-add-script` — inherits `.btn .btn-primary` accent styling (#ff6b35 background), drawing the eye as the entry point for the section.

### Per-item structure (rendered by JS)

```
Script 1                          [↑] [↓] [×]
┌─────────────────────────────────────────────┐
│ <script>                                    │
│   // seu código aqui                        │
│ </script>                                   │
└─────────────────────────────────────────────┘
```

Each `.script-item` contains:
1. `.script-item-header` row: label + `.script-item-controls` (↑, ↓, × buttons)
2. `<textarea class="mono">` with placeholder and `rows="4"` (min-height 80px via existing rule)

### "Adicionar Script" button

Uses `.btn .btn-primary` with reduced padding override: `padding: 8px 16px; font-size: 0.85rem`. Positioned below the script list with `margin-top: 12px`. Always visible once the section is shown.

## Interaction Contract

### Add (SCRIPTS-01)

- Click "**+ Adicionar Script**" appends a new `.script-item` to `#scripts-list-container`.
- New item gets an empty textarea, auto-numbered label ("Script N"), and ↑/↓/× buttons.
- `state.extraScripts` receives a new entry: `{ id: crypto.randomUUID(), content: '' }` (object with `id` to simplify reorder, content synced on export read).
- After add: focus moves to the new textarea automatically.
- Reorder buttons on the new item: ↑ disabled if it is the first item, ↓ disabled (it is always the last).

### Remove (SCRIPTS-02)

- Click × on any item: remove that `.script-item` from DOM, remove matching entry from `state.extraScripts` by id.
- After removal: re-render labels ("Script 1", "Script 2"…) and update ↑/↓ disabled states for the now-first and now-last items.
- No confirmation dialog — removal is low-stakes (user can re-add).

### Reorder (SCRIPTS-03)

- Click ↑: swap item with the one above it in both DOM and `state.extraScripts` array.
- Click ↓: swap item with the one below in both DOM and `state.extraScripts` array.
- After swap: update disabled states — first item's ↑ disables, last item's ↓ disables; all others enabled.
- ↑ of first item: button is disabled (`disabled` attribute), click does nothing (D-11).
- ↓ of last item: button is disabled (`disabled` attribute), click does nothing (D-11).
- No animation — items swap positions immediately (matches project's zero-animation approach for functional interactions).

### Content sync for export

- `state.extraScripts` is an array of `{ id, content }` objects in display order.
- Before building the export payload, read each textarea value into its matching `state.extraScripts` entry by `data-script-id` attribute on the textarea.
- Export payload sends `extraScripts: state.extraScripts.map(s => s.content)` — string array in order.

### Section reveal

- `showSection(sectionExtraScripts)` called unconditionally in the fetch success callback (D-03).
- Section always visible after any successful fetch, even with empty list.

### Auto-wrap (D-14)

Handled server-side in `buildExportHtml()`. Frontend sends content as-is; server checks if content starts with `<script` (case-insensitive) and wraps if needed. No client-side indication needed.

## Export Payload Contract

```js
// In export handler, add to payload object:
extraScripts: state.extraScripts.map(function(s) {
  // Read current textarea value for each item before mapping
  var ta = document.querySelector('textarea[data-script-id="' + s.id + '"]');
  return ta ? ta.value : s.content;
}),
```

## Responsive Behavior

- Section card fills container width (max 860px) — no change from existing behavior.
- Script items stack vertically — no grid layout needed.
- `.script-item-header`: on mobile (< 600px), label and button group remain on the same row. Button group is `flex-shrink: 0` so it never wraps. Label truncates with `overflow: hidden; text-overflow: ellipsis; white-space: nowrap` if needed.
- "Adicionar Script" button: full-width on mobile using `@media (max-width: 600px) { #btn-add-script { width: 100%; justify-content: center; } }`.
