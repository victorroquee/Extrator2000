Add the `buildElementorJson(html)` function to server.js, placed AFTER `buildExportHtml` (after line ~1293) and BEFORE the route definitions (before line ~1295 `// ── Route: POST /api/export`).

The function takes one argument: `html` (string) — the fully-baked affiliate HTML output from `buildExportHtml()`.

**Implementation steps:**

1. **Parse HTML with cheerio** (per existing codebase pattern):
   ```javascript
   const $ = cheerio.load(html, { decodeEntities: false });
   ```

2. **Extract page title** (per D-09):
   ```javascript
   const rawTitle = $('title').first().text().trim() || '';
   ```

3. **Generate unique ID helper** (per D-05, Pitfall 1):
   ```javascript
   const usedIds = new Set();
   function genId() {
     let id;
     do { id = crypto.randomBytes(4).toString('hex'); } while (usedIds.has(id));
     usedIds.add(id);
     return id;
   }
   ```
   This helper is local to each `buildElementorJson` call. The `usedIds` Set prevents collisions (Pitfall 1).

4. **Build head container** (per D-03, D-04):
   - Extract all content from `$('head')` — scripts, styles, link tags
   - Filter out `<title>` and `<meta charset>` tags (Elementor manages its own)
   - Collect the remaining innerHTML as a single string
   - If non-empty, create the first container:
     ```javascript
     {
       id: genId(),
       elType: 'container',
       isInner: false,
       settings: { flex_direction: 'column' },
       elements: [{
         id: genId(),
         elType: 'widget',
         widgetType: 'html',
         isInner: false,
         settings: { html: headContent },
         elements: []
       }]
     }
     ```

5. **Determine body sections** (per D-01, D-02):
   - Get direct children of `<body>`: `const bodyChildren = $('body').children();`
   - If `<body>` has exactly 1 child element (wrapper div), look one level deeper: use that wrapper's children instead (D-02)
   - Filter out text nodes, comments, and `<script>` tags at body level (Claude's discretion from CONTEXT.md)
   - Each remaining child element becomes a separate top-level container

6. **Build section containers** (per D-07, D-08):
   For each body section element:
   ```javascript
   {
     id: genId(),
     elType: 'container',
     isInner: false,
     settings: { flex_direction: 'column' },
     elements: [{
       id: genId(),
       elType: 'widget',
       widgetType: 'html',
       isInner: false,
       settings: { html: $.html(sectionElement) },
       elements: []
     }]
   }
   ```
   - Container `settings` is always `{}` object, never `[]` array (Pitfall 8)
   - `flex_direction: 'column'` is always explicit (Pitfall 13)
   - `isInner: false` for all top-level containers (Pitfall 2)
   - Widget `isInner: false` matches reference file pattern

7. **Assemble root JSON envelope** (per D-06, Pitfalls 15, 16):
   ```javascript
   return {
     version: '0.4',
     title: rawTitle,
     type: 'page',
     page_settings: {},
     content: containers
   };
   ```
   - `type` MUST be `'page'` (Pitfall 15)
   - `page_settings` MUST be `{}` (Pitfall 16), never `null` or `[]`

8. **Add module.exports** at the bottom of server.js (after existing exports at line ~1613):
   ```javascript
   module.exports.buildElementorJson = buildElementorJson;
   ```

**Edge cases to handle:**
- Empty `<head>` content after filtering: skip the head container entirely (no empty container)
- `<body>` with no children: return valid JSON with empty `content: []`
- Malformed HTML input: cheerio handles gracefully; no additional error handling needed
- `$.html(element)` preserves the outer tag of the element (the full section markup)

**What NOT to do:**
- Do NOT attempt full DOM-to-widget decomposition (Anti-Pattern 2 from ARCHITECTURE.md) — use `html` widget for all content
- Do NOT include the full `<html><head>...</head><body>...</body></html>` in a single widget (Anti-Pattern 3)
- Do NOT use sequential/patterned IDs like `el_0001` (Anti-Pattern 4) — use `crypto.randomBytes(4).toString('hex')`
- Do NOT use `settings: []` (array) anywhere — always `settings: {}` (object) (Pitfall 8)
