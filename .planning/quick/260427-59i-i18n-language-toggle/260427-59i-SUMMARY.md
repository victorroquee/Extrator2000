# Quick Task 260427-59i: i18n Language Toggle (PT-BR/EN)

## What was done

1. **server.js**: Added GET `/us` route serving same `index.html`
2. **public/index.html**: Full i18n system with:
   - Translations dictionary (`T`) with ~100 keys for both PT-BR and EN
   - `t()` helper with fallback to PT-BR
   - `applyLang()` function for `data-i18n`, `data-i18n-placeholder`, `data-i18n-title`
   - 61 `data-i18n` attributes, 6 `data-i18n-placeholder`, 1 `data-i18n-title`
   - Flag toggle (🇧🇷/🇺🇸) in navbar, discrete, next to version badge
   - All dynamic JS strings replaced with `t()` calls
   - Auto-detection of `/us` path on page load

## Files changed

- `server.js` — +5 lines (route)
- `public/index.html` — +498/-153 lines (i18n system + data attributes + string replacements)

## How it works

- `/` → PT-BR (default)
- `/us` → English
- Flag toggle in navbar navigates between routes
- `LANG` variable set from `window.location.pathname` on load
- `applyLang()` updates all `data-i18n` elements and sets `html[lang]`
